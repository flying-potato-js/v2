## How to Contribute

There are many ways to contribute:
- Reporting bugs
- Suggesting features or enhancements
- Improving documentation
- Contributing code or examples

Check existing issues and pull requests first to avoid duplication. Discuss ideas in an issue before starting!

## Code Style Guide

- Follow the existing JavaScript and documentation style.
- Write clear, maintainable code with comments.
- Keep pull requests focused; avoid unrelated changes.
- Use meaningful commit messages.

## Reporting Issues

- Check if your issue is already reported.
- Provide a descriptive title and details (steps to reproduce, expected/actual behavior, and environment info).
- Include screenshots or code snippets if relevant.

## Pull Requests

- Fork the repo and create a branch for your changes.
- Ensure your code passes all tests and add new ones if needed.
- Reference related issues with keywords (e.g., `Fixes #123`).
- Fill out the PR template with a summary and context.
- Respond to feedback and update your PR as needed.
